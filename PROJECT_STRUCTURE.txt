KEY FILE PURPOSES
═════════════════

ENTRY POINT:
  src/server.ts
    - Creates Express app
    - Sets up webhook endpoint (POST /webhook)
    - Validates signatures & rate limits
    - Returns 202, processes async
    - Handles errors gracefully

CORE TRADING:
  src/strategies/position-flipper.ts
    - Receives TradingView alert
    - Fetches current position
    - Closes opposite position (if exists)
    - Opens new position
    - Returns timing metrics

EXCHANGE CLIENT:
  src/exchange/hyperliquid.ts
    - HTTP client to Hyperliquid API
    - Methods: getPosition, placeMarketOrder, closePosition, flipPosition
    - Connection pooling for performance
    - Timeout handling

SECURITY:
  src/security/auth.ts
    - HMAC-SHA256 signature verification
    - Prevents webhook spoofing

  src/security/validator.ts
    - Zod schema validation
    - Rate limiting (per-symbol)
    - Order size validation
    - Leverage validation

CONFIGURATION:
  src/config/config.ts
    - Loads environment variables
    - Validates required settings
    - Separates testnet/mainnet
    - Returns typed config object

UTILITIES:
  src/utils/logger.ts
    - Pino logger with pretty-printing in dev
    - Structured logging in production

  src/utils/error-handler.ts
    - Custom error types (BotError, APIError, ValidationError, etc.)
    - Error severity classification

  src/utils/metrics.ts
    - Tracks webhooks received/processed
    - Records latency metrics
    - Records positions opened/closed

PERFORMANCE LAYOUT
═══════════════════

Request Flow (sub-50ms typical):

1. TradingView → HTTP POST
2. server.ts receives webhook
   └─ Extract JSON payload
3. security/auth.ts
   └─ Verify HMAC signature (~1ms)
4. security/validator.ts
   └─ Validate schema, rate limit (~2ms)
5. position-flipper.ts
   └─ Get current position (~5ms)
6. exchange/hyperliquid.ts (parallel)
   ├─ Close old position (~15ms)
   └─ Open new position (~15ms)
7. Return result & log

Total: 20-50ms (no sleep delays!)

  1. Start in src/strategies/position-flipper.ts
  2. Use src/exchange/hyperliquid.ts for API calls
  3. Add validation in src/security/validator.ts
  4. Update types in src/types/alert.ts
  5. Test with ./test-webhook.sh
